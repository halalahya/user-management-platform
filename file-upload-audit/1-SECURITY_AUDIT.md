# 头像上传模块安全审计报告

**审计范围：** Flask用户管理系统 — 头像上传功能（/upload路由）  
**审计日期：** 2026年7月  
**漏洞数量：** 共发现 **4 项安全漏洞**

---

## 漏洞总览

| 编号 | 漏洞名称 | 风险等级 | 影响 |
|------|---------|---------|------|
| VUL-FU-01 | 任意文件上传（无后缀名校验） | 高危 | 服务器被上传 Webshell、恶意脚本 |
| VUL-FU-02 | 文件覆盖（同名文件互相覆盖） | 中危 | 用户头像被恶意替换 |
| VUL-FU-03 | 无文件内容真实性校验 | 中危 | 非图片文件伪装上传，存储空间浪费 |
| VUL-FU-04 | 文件体积上限过大 | 低危 | 16MB 上限可被用于存储耗尽攻击 |

---

## 漏洞详情

### VUL-FU-01：任意文件上传（高危）

**漏洞代码位置：** `app.py` 第307-308行

```python
filename = secure_filename(file.filename)
file.save(os.path.join(UPLOAD_FOLDER, filename))
```

**漏洞成因：** 上传接口未对文件的扩展名、MIME类型做任何检查。用户上传的任何文件（.php、.py、.jsp、.html、.svg、.exe）都会被原样保存到 `static/uploads/` 目录，并且可以直接通过 URL 访问。

**攻击利用方式：**

攻击者登录后，使用 curl 或 Burp Suite 上传一个 Webshell 脚本（Python 后门），通过浏览器访问获取执行权限。

```
# 上传恶意 Python 脚本
curl -X POST http://127.0.0.1:5000/upload \
  -b "session=..." \
  -F "file=@shell.py"

# 访问后门执行命令
curl http://127.0.0.1:5000/static/uploads/shell.py?cmd=whoami
```

**危害说明：** 攻击者上传包含恶意代码的脚本文件后，一旦服务器支持脚本执行（如 Apache + PHP、Flask 静态目录可解析 .py），即可获得服务器远程控制权限，进一步实施内网渗透、数据窃取、勒索软件部署等攻击。即使不支持执行，上传 .html 或 .svg 文件也可用于 XSS 钓鱼攻击。

**风险等级：** 高危（CVSS 3.1 Score: 8.2）

---

### VUL-FU-02：文件覆盖（中危）

**漏洞代码位置：** `app.py` 第307-308行

```python
filename = secure_filename(file.filename)
file.save(os.path.join(UPLOAD_FOLDER, filename))
```

**漏洞成因：** 使用用户提供的原始文件名直接保存，未添加任何唯一性标识。两名用户上传相同文件名的文件时，后上传的文件会直接覆盖前者。

**攻击利用方式：** 攻击者上传一个与目标用户头像同名的恶意文件，覆盖其正常头像文件，达到篡改或植入恶意内容的目的。

**危害说明：** 用户头像被恶意替换，导致身份冒用或恶意内容传播。同时，同名文件的覆盖也会导致正常用户的头像丢失。

**风险等级：** 中危（CVSS 3.1 Score: 4.3）

---

### VUL-FU-03：无文件内容真实性校验（中危）

**漏洞代码位置：** `app.py` 第303-308行

```python
file = request.files.get("file")
if not file or file.filename == "":
    return render_template("upload.html", error="请选择一个文件")
filename = secure_filename(file.filename)
file.save(os.path.join(UPLOAD_FOLDER, filename))
```

**漏洞成因：** 仅检查了文件是否存在，未对文件内容做任何验证。将一个 .txt 文件重命名为 .png 上传也会被保存，系统无法判断其是否为一真正的图片文件。

**攻击利用方式：** 攻击者上传一个伪装成图片的恶意代码文件，利用某些浏览器或解析器的 MIME 嗅探行为执行恶意内容。

**危害说明：** 存储空间被无意义数据占用；恶意内容被伪装成图片文件传播，增加社会工程攻击的成功率。

**风险等级：** 中危（CVSS 3.1 Score: 5.3）

---

### VUL-FU-04：文件体积上限过大（低危）

**漏洞代码位置：** `app.py` 第26行

```python
app.config["MAX_CONTENT_LENGTH"] = 16 * 1024 * 1024
```

**漏洞成因：** 16MB 的上传上限对于头像图片来说过大，正常头像图片通常不超过 1-2MB。过大的上限允许攻击者批量上传大文件，快速耗尽服务器存储空间。

**攻击利用方式：** 攻击者循环上传 16MB 的垃圾文件，填满磁盘分区，导致服务不可用。

**危害说明：** 磁盘空间耗尽，服务器无法写入新数据，导致文件上传、数据库写入等功能瘫痪。

**风险等级：** 低危（CVSS 3.1 Score: 3.3）

---

## 审计结论

头像上传模块共发现 4 项安全漏洞，其中高危 1 项（任意文件上传可直接导致远程代码执行），中危 2 项，低危 1 项。修复的核心在于增加文件扩展名白名单校验、文件内容真实性校验以及文件名唯一化处理。
